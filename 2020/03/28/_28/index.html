# JavaScript之微任务与宏任务

JavaScript作为浏览器脚本语言，主要用途是与用户交互，以及操作DOM。所以决定了它只能是单线程语言，即同一个时间只能做一件事。为了利用多核CPU的计算能力，虽然HTML5提出了Web Worker，允许JavaScript脚本创建多个线程，但是子线程完全受主线程控制，且不得操作DOM和BOM。所以，依然没有改变JavaScript是单线程的本质。 为了解决单线程导致的线程等待资源，cpu空闲，而其他任务一直等待的问题。将所有的任务分为两种，一种是同步任务，一种是异步任务。同步任务指的是，在主线程上排队执行的任务，只有前一个任务执行完毕，才能执行下一个任务。异步任务指的是，不进入主线程，而进入“任务队列”的任务，自由“任务队列”通知主线程，某个异步任务可以执行了，该任务才会进入主线程执行。 主线程与任务队列执行关系如下：

# 微任务与宏任务

在任务队列中，又分为微任务与宏任务。

  • 宏任务：I/O 、setTimeout、setInterval；
  • 微任务：Promise.then catch finally、process.nextTick。

接下来通过几个面试题来深入理解一下主线程与任务队列的执行机制，代码如下：

  console.log(1);
  setTimeout(function(){
    console.log(2);
  }, 500);
  new Promise((resolve, rejected) => {
    console.log(3);
    resolve();
  }).then(function(){
    console.log(4);
  })
  console.log(5);

在浏览器中运行会得到如下结果： 1 3 5 4 2。现在我们来分析一下这段代码，首先我们知道JavaScript从上到下的执行顺序，所以直接执行第一句，打印出1；接下来解析到setTimeout，被作为宏任务加入到异步任务队列中，然后执行到Promie，在这里我们要记住Promise的内容跟主线程一样，直接执行，所以打印出3；然后进入then部分，then被作为微任务加入到任务队列，接下来便是执行到后面的打印5；到此为止，主线程便执行完成，接下来我们来看任务队列，在队列中存在1个宏任务、1个微任务，这个时候会先执行微任务，再去执行宏任务，所有先打印4，最后打印2。

经过以上的例子，大家应该对微任务与宏任务有了一定的了解，接下里我们再来看一个更复杂的例子，代码如下：

  console.log("a");
  setTimeout(function(){
    console.log("b");
    new Promise((resolve, rejected) => {
      console.log("c");
      resolve();
    }).then(function(){
      console.log("d");
    })
  }, 500);
  new Promise((resolve, rejected) => {
    console.log("e");
    resolve();
  }).then(function(){
    console.log("f");
    setTimeout(function(){
      console.log("g");
    }, 500)
  })
  console.log("h");

这次这个就比较复杂了，我们好好来梳理一下，运行时候首先打印出a，然后遇到setTimeout，被作为宏任务加入任务队列，这里我们假设宏任务队列类似一个数组的样子，我们称它为macro,那此刻它就是[setTimeout1],接下来继续往下执行到Promise打印出e,同样我们假设微任务队列也类似一个数组的样子，称它为micro，然后then被作为微任务加入到micro，即[then1]；最后主线程执行到最后一打印出h，到此为止主线程变执行完毕，接下来我们来看任务队列，队列中包含微任务队列和宏任务队列，我们知道应该先清空微任务队列，所以先执行micro里的then1，便先打印出f,然后遇到里面的setTimeout，此时setTimeout又被作为宏任务加入到macro,那macro便是[setTimeout1, setTimeout2];这个时候微任务队列已被清空，那便该执行宏任务队列，按照顺序先执行setTimeout1，首先打印b，然后执行Promise打印c，最后then被作为微任务加入到micro，此时micro为[then2]，到此setTimeout1这个宏任务便执行完毕，我们知道每个宏任务执行完毕后首先应该检查微任务队列是否存在待执行任务，如果有则需要等微任务队列依序执行完毕后方可执行下一个宏任务，而此时的micro存在微任务then2，所以便执行then2，打印d，这样微任务队列被清空，那便执行下一个宏任务setTimeout2，打印出g，这样整段代码便执行完毕，结果如下：a e h f b c d g。

敲黑板划重点：

# 从小到大，先依序清空微任务再执行下一个宏任务，每个宏任务结束以后，需要先清空微任务队列再执行下一个宏任务。
